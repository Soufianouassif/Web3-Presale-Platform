## تهيئة عقد البيع (Initialize)

بعد النشر، استدعِ تعليمة `initialize` مرة واحدة فقط:

```typescript
// الأسعار (4 مراحل)
// المرحلة 1: 0.00000001$ للرمز → 100M رمز لكل 1 USDT
// المرحلة 2: 0.00000002$ للرمز → 50M رمز لكل 1 USDT
// المرحلة 3: 0.00000004$ للرمز → 25M رمز لكل 1 USDT
// المرحلة 4: 0.00000006$ للرمز → ~16.7M رمز لكل 1 USDT

await program.methods.initialize({
  treasury: TREASURY_WALLET_PUBKEY,        // محفظة SOL
  usdtTreasuryAta: USDT_ATA_PUBKEY,        // ATA لاستقبال USDT-SPL
  usdtMint: new PublicKey("Es9vMFrzaCERmJfrF4H2FYD4KCoNkY11McCe8BenwNYB"), // USDT Mainnet
  presaleStart: new BN(Math.floor(Date.now() / 1000)),       // الآن
  presaleEnd:   new BN(Math.floor(Date.now() / 1000) + 86400 * 30), // بعد 30 يوم
  claimOpensAt: new BN(Math.floor(Date.now() / 1000) + 86400 * 60), // بعد 60 يوم
  solPriceUsdE6: new BN(150_000_000), // سعر SOL = $150 (عدله حسب سعر السوق)
}).rpc();
```

## أوامر الأدمن اليومية

### تشغيل البيع وإيقافه
```bash
# تفعيل البيع (يجب استدعاؤها قبل أي شراء)
anchor run activate

# إيقاف مؤقت لحالة طوارئ
anchor run pause

# استئناف بعد الإيقاف
anchor run resume

# إنهاء البيع نهائياً
anchor run end_presale
```

### تحديث سعر SOL
```typescript
// عدّل السعر يومياً أو عند تغيير كبير في سعر SOL
await program.methods.updateSolPrice(
  new BN(180_000_000) // $180 بـ 6 decimals
).rpc();
```

### التخصيص اليدوي (ERC20 / TRC20)
```typescript
// عندما يدفع شخص عبر Ethereum أو TRON
// 1. تحقق من المعاملة يدوياً على Etherscan / Tronscan
// 2. احصل على عنوان محفظة Solana الخاصة بهم
// 3. استدعِ هذه التعليمة:

const refBuffer = Buffer.alloc(64).fill(0);
Buffer.from("ETH_TX_0xabc123...").copy(refBuffer);

await program.methods.manualAllocate({
  buyerWallet: new PublicKey("BUYER_SOLANA_WALLET"),
  tokens: new BN(1_000_000_000), // عدد الرموز كأرقام صحيحة (بدون decimals)
  reference: Array.from(refBuffer), // hash المعاملة للمرجعية
}).rpc();
```

## توزيع الكليم

- **20%** تُفتح فور فتح الكليم (TGE)
- **80%** الباقية تُوزَّع على 4 أشهر (20% شهرياً)
- المستخدم يستدعي `claim_tge` مرة واحدة
- المستخدم يستدعي `claim_vesting` كل شهر (حتى 4 مرات)

## إضافة ميزات مستقبلية

### قائمة بيضاء (Whitelist)
- استخدم `_reserved` bytes في `PresaleConfig`
- أضف Merkle root في الـ reserved space
- أضف تعليمة `buy_whitelisted` تتحقق من Merkle proof

### نظام الإحالة (Referral)
- أضف `referrer: Option<Pubkey>` إلى BuyerRecord
- أضف `referral_bonus_percent: u8` في PresaleConfig
- احسب المكافأة عند كل شراء

### التحقق التلقائي من مراحل متعددة
- الانتقال بين المراحل تلقائي عند بيع كل مرحلة بالكامل
- يمكن الانتقال اليدوي عبر `advance_stage`

## ملاحظات أمنية مهمة

1. **احتفظ بـ seed phrase** محفظة الأدمن في مكان آمن دون اتصال بالإنترنت
2. **لا تشارك** مفتاحك الخاص مع أحد
3. **اختبر على Devnet** قبل النشر على Mainnet دائماً
4. **عقد البيع** لا يحتفظ بالأموال — SOL يذهب مباشرة للـ treasury
5. **عقد الكليم** يحتاج تعبئة الـ vault بـ $PWIFE قبل فتح الكليم
6. **سعر SOL** يحتاج تحديثاً يدوياً — فكّر في ربطه بـ oracle مستقبلاً
