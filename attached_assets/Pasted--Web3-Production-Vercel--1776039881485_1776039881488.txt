أنت تعمل على مشروع Web3 في بيئة إنتاجية (Production) مستضاف على Vercel.

⚠️ قاعدة مهمة:
لا تقم بإعادة بناء المشروع من الصفر.
لا تغيّر واجهة المستخدم بدون داعٍ.
لا تكسر أي منطق يعمل حالياً.
لا تعدّل منطق المشروع إلا إذا كان ذلك ضروريًا للأمان.
قم فقط بإصلاحات أمنية دقيقة ومحددة.

1. فرض إعادة الموافقة على ربط المحفظة بعد تسجيل الخروج

المشكلة الحالية:
بعد تسجيل الخروج، قد يتم إعادة ربط المحفظة بسهولة بسبب بقاء بيانات الجلسة أو حالة المحفظة محفوظة.

السلوك المطلوب:

* يجب على المستخدم إعادة ربط المحفظة يدوياً بعد تسجيل الخروج
* منع أي إعادة اتصال تلقائي (auto reconnect)
* عدم الدخول مباشرة إلى dashboard بعد تسجيل الخروج
* عدم الاعتماد على بيانات جلسة قديمة
* يجب طلب موافقة المستخدم من جديد

المطلوب تنفيذه:

* فصل المحفظة بالكامل عند تسجيل الخروج
* حذف كل البيانات المرتبطة بالمحفظة من localStorage و sessionStorage
* حذف أي بيانات محفوظة في wallet adapter
* إعادة تعيين حالة المستخدم والمحفظة (React state أو store)
* إعادة توجيه المستخدم إلى صفحة /connect أو الصفحة الرئيسية
* منع إعادة الاتصال التلقائي بعد تحديث الصفحة
* التأكد أن autoConnect غير مفعل

يجب دعم:

* Phantom
* Solflare
* أي محفظة Solana مستخدمة

━━━━━━━━━━━━━━━━━━━

2. تقوية التحقق من جلسة المحفظة

لا تعتمد على sessionStorage أو localStorage فقط.

يجب حماية الصفحات:

* /dashboard
* صفحات المستخدم
* أي صفحات محمية (staking / presale / claim)

يجب التحقق باستخدام:

* window.solana.isConnected
* publicKey الحقيقي من المحفظة
* مقارنة العنوان المخزن مع العنوان الحقيقي

القاعدة:
إذا كانت البيانات المخزنة موجودة ولكن المحفظة غير متصلة فعليًا → اعتبر المستخدم غير مسجل الدخول.

━━━━━━━━━━━━━━━━━━━

3. إصلاح مشكلة CORS

المشكلة:
الهيدر يرجع:
Access-Control-Allow-Origin: *

المطلوب:

* البحث عن:
  cors(
  Access-Control-Allow-Origin
  res.setHeader

* استبدال النجمة '*' بـ:
  https://pwifecoin.fun

* عدم كسر الاتصال بين الواجهة والـ API

━━━━━━━━━━━━━━━━━━━

4. الحفاظ على أمان CSP

* لا تضعف سياسة CSP
* حافظ على الإعدادات الحالية أو حسنها
* لا تضف unsafe إلا إذا كان ضروريًا جدًا

━━━━━━━━━━━━━━━━━━━

5. التحقق من أمان تسجيل الخروج

بعد logout يجب التأكد من:

* تم فصل المحفظة فعليًا
* تم حذف sessionStorage
* تم حذف localStorage
* لا يمكن الدخول إلى /dashboard مباشرة
* تحديث الصفحة لا يعيد الاتصال
* لا يمكن التلاعب بالجلسة عبر Console
* تغيير المحفظة يتم التعامل معه بشكل صحيح

━━━━━━━━━━━━━━━━━━━

6. تحسينات أمنية إضافية

* حماية قوية للـ routes
* إذا كان الشك في الجلسة → إعادة التوجيه إلى connect
* عدم الثقة في البيانات القادمة من frontend فقط
* منع استرجاع جلسة قديمة
* logout يجب أن يكون آمن حتى لو تم الضغط عليه عدة مرات

━━━━━━━━━━━━━━━━━━━

7. لا تكسر هذه الأشياء

لا تكسر:

* نظام presale
* dashboard
* ربط المحافظ
* routing
* Solana RPC
* CoinGecko API
* نظام admin
* تصميم الموقع

━━━━━━━━━━━━━━━━━━━

8. طريقة التنفيذ

* استعمل نفس هيكلة المشروع
* قم بتعديلات صغيرة فقط
* لا تعيد كتابة النظام بالكامل
* لا تغير ملفات غير ضرورية

━━━━━━━━━━━━━━━━━━━

9. بعد التنفيذ، تحقق من:

* تسجيل الدخول يعمل
* تسجيل الخروج يفصل الجلسة بالكامل
* refresh لا يعيد الاتصال
* /dashboard محمية
* session المزورة لا تعمل
* تغيير المحفظة يعمل بشكل صحيح
* CORS لم يعد '*'
* الموقع يعمل بدون مشاكل
