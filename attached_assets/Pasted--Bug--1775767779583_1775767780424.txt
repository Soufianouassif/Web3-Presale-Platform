أريد الآن تنفيذ إصلاح شامل ونهائي للمشروع بالكامل، لأن المشكل لم يعد Bug بسيط، بل هو مزيج من:

- نظام إحالة غير مكتمل أو غير مربوط end-to-end
- واجهات قد تكون static أو mock
- اعتماد سابق على client values
- وجود DEV MODE
- احتمال mismatch في قاعدة البيانات بين Replit و Vercel (Neon)
- ضرورة ربط النظام بالبلوكشين الحقيقي (Devnet)
- الحاجة إلى أمان Production حقيقي

مهم جداً:
- لا أريد إصلاح جزئي
- لا أريد افتراض أن النظام يعمل
- أريد إعادة بناء وربط كل شيء end-to-end
- أريد إثبات عملي أن كل شيء يعمل فعلاً

🧩 المرحلة 1 — توحيد قاعدة البيانات (أهم خطوة)
━━━━━━━━━━━━━━━━━━

افحص وحدد بدقة:

- DATABASE_URL في Replit
- DATABASE_URL في Vercel
- هل كلاهما يستعمل Neon أم لا
- هل هناك DB مدمجة داخل Replit تُستعمل بدون قصد

المطلوب:

1. توحيد source of truth:
   → كل البيئات تستعمل Neon database

2. إزالة أي fallback لقاعدة محلية

3. التأكد أن:
   - purchases
   - referrals
   - referral_codes
   - users

كلها موجودة في Neon وبنفس schema

4. تطبيق migrations على Neon

5. التأكد أن:
   - backend يكتب في Neon
   - dashboard يقرأ من Neon
