أواجه مشكلاً كبيراً في نظام الإحالة داخل موقعي، وأريد منك عمل Debug شامل وحقيقي من البداية إلى النهاية، وليس مجرد افتراض أن المشكلة تم إصلاحها.

المشكلة الحالية:
- أدخل إلى الموقع عبر رابط إحالة
- أقوم باختبار الشراء
- لا أعلم هل الإحالة يتم احتسابها فعلاً أم لا
- لا يتم تحديث بيانات الإحالة في لوحة المحيل
- لا يتم احتساب مكافأة المحيل
- لا تظهر بيانات الإحالة في لوحة الأدمن
- سبق أن قلت إنك أصلحت المشكلة، لكن عند الاختبار لم يتغير أي شيء

مهم جداً:
لا تفترض أن أي جزء من نظام الإحالة يعمل.
قم بفحص النظام كاملاً end-to-end خطوة بخطوة.

الهدف
━━━━━━━━━━━━━━━━━━

أريد منك تتبع مسار الإحالة بالكامل من هذه المراحل:

1. دخول المستخدم عبر رابط إحالة
2. حفظ referral code أو referrer identifier
3. بقاء referral data محفوظة حتى بعد التنقل داخل الموقع
4. إرسال referral data أثناء عملية الشراء
5. تسجيل عملية الشراء في قاعدة البيانات
6. ربط عملية الشراء بالمحيل الصحيح
7. إنشاء referral record
8. احتساب reward أو commission
9. تحديث لوحة المحيل
10. تحديث لوحة الأدمن
11. التأكد أن البيانات تظهر مباشرة أو بعد refresh بشكل صحيح

المرحلة 1 — تتبع دخول رابط الإحالة
━━━━━━━━━━━━━━━━━━

تحقق من:
- كيف يتم استقبال referral code من URL
- هل يتم حفظه في localStorage أو cookie أو session أو state
- هل يبقى محفوظاً بعد الانتقال بين الصفحات
- هل يضيع عند refresh أو عند الانتقال إلى صفحة الشراء
- أضف logs واضحة جداً لمعرفة:
  - هل referral code وصل من الرابط
  - أين تم حفظه
  - هل بقي محفوظاً حتى وقت الشراء

إذا كان referral code لا يُحفظ أو يضيع:
- أصلح ذلك مباشرة

━━━━━━━━━━━━━━━━━━
المرحلة 2 — تتبع وقت الشراء
━━━━━━━━━━━━━━━━━━

افحص purchase flow بالكامل:

- عند الضغط على buy:
  - هل referral code يتم إرساله مع الطلب؟
  - هل purchase request تحتوي فعلاً على referrer data؟
  - هل backend يستقبلها؟
  - هل backend يتجاهلها أو يفقدها؟

أضف logs دقيقة في:
- frontend قبل إرسال الطلب
- request payload
- backend عند استقبال الطلب
- قبل الحفظ في قاعدة البيانات
- بعد الحفظ في قاعدة البيانات

إذا كان referral code لا يصل إلى backend:
- أصلح نقطة الربط

━━━━━━━━━━━━━━━━━━
المرحلة 3 — قاعدة البيانات
━━━━━━━━━━━━━━━━━━

افحص الجداول المرتبطة مثل:
- purchases
- referrals
- referral_rewards
- users / investors / wallets
- أي جدول له علاقة بالمحيل أو المكافآت

أريد منك أن تتحقق فعلياً:

1. هل purchase يتم حفظها؟
2. هل purchase تحتوي على referrerId أو referralCode أو linked referrer؟
3. هل يتم إنشاء سجل referral بعد الشراء؟
4. هل reward يتم حسابها وتخزينها؟
5. هل هناك transaction أو query تفشل بصمت؟
6. هل هناك field names غير متطابقة بين الكود و DB؟
7. هل هناك migration ناقصة أو column غير موجود أو null دائماً؟

إذا وجدت أي mismatch بين الكود وقاعدة البيانات:
- أصلحه مباشرة

━━━━━━━━━━━━━━━━━━
المرحلة 4 — منطق احتساب المكافأة
━━━━━━━━━━━━━━━━━━

تحقق من منطق reward calculation:

- متى يتم احتساب reward؟
- هل يتم فقط إذا purchase status = completed؟
- هل status لا تتغير إلى completed؟
- هل هناك شرط يمنع reward بدون قصد؟
- هل يتم تجاهل self-referral؟
- هل purchaseId مطلوب ولكن لا يتم التحقق منه جيداً؟
- هل reward يُحسب لكن لا يُحفظ؟
- هل يُحفظ لكن dashboard لا يقرأه؟

أريد تتبع دقيق:
- input
- calculation
- save
- display

إذا كان reward calculation broken:
- أصلحه

المرحلة 5 — لوحة المحيل Dashboard
━━━━━━━━━━━━━━━━━━

افحص Dashboard الخاص بالمحيل:

- من أين تأتي بيانات referrals؟
- من أي endpoint؟
- من أي query؟
- هل endpoint يعمل فعلاً؟
- هل يرجع بيانات فارغة؟
- هل يعتمد على wallet أو userId بشكل خاطئ؟
- هل هناك cache أو stale state يمنع التحديث؟
- هل البيانات تحتاج refresh يدوي؟
- هل query تربط wallet غير صحيحة؟

أريد إصلاح:
- total referrals
- total referred purchases
- total rewards
- referral history
- latest referred users/purchases

المرحلة 6 — لوحة Admin
━━━━━━━━━━━━━━━━━━

افحص لوحة الأدمن:

- هل admin endpoint يرجع referral data؟
- هل query admin تجلب referrals فعلاً؟
- هل هناك filter أو join خاطئ؟
- هل البيانات مخزنة لكن لا تظهر؟
- هل column names غير متطابقة؟
- هل لوحة الأدمن تعرض من source خاطئ؟

أصلح عرض:
- جميع الإحالات
- اسم أو wallet المحيل
- اسم أو wallet المشتري عبر الإحالة
- قيمة الشراء
- قيمة المكافأة
- تاريخ العملية
- حالة الإحالة

المرحلة 8 — اختبار حقيقي End-to-End
━━━━━━━━━━━━━━━━━━

أريد منك تنفيذ أو إعداد اختبار حقيقي للمسار التالي:

1. افتح الموقع مع referral code
2. تأكد أنه تم حفظه
3. نفذ شراء تجريبي
4. تحقق من السجلات في DB
5. تحقق من إنشاء referral record
6. تحقق من reward record
7. تحقق من لوحة المحيل
8. تحقق من لوحة الأدمن

إذا كان الاختبار لا يمكن تنفيذه تلقائياً:
- أضف صفحة debug مؤقتة أو logs كافية جداً
- وأعطني تقريراً واضحاً أين تتعطل السلسلة بالضبط

المرحلة 9 — المطلوب النهائي
━━━━━━━━━━━━━━━━━━

أريد منك في النهاية:

1. تحديد السبب الحقيقي للمشكلة بدقة
2. تحديد هل الخلل في frontend أو backend أو database أو dashboards
3. إصلاحه فعلياً في الكود
4. عدم الاكتفاء بقول "تم الإصلاح"
5. إعطائي تقرير واضح:
   - ما الذي كان مكسوراً
   - ما الملفات التي تم تعديلها
   - ما الذي أصبح يعمل الآن
   - كيف أختبره بنفسي خطوة بخطوة

قواعد صارمة
━━━━━━━━━━━━━━━━━━

- لا تفترض أن النظام يعمل
- لا تعطيني وعوداً عامة
- لا تقل "تم الإصلاح" إلا بعد تتبع المسار كاملاً
- أضف logs وتشخيص حقيقي
- أصلح root cause وليس الأعراض
- إذا وجدت أكثر من خلل، أصلحها كلها
- حافظ على المنطق الحالي للموقع ولا تكسر الشراء أو الواجهة
