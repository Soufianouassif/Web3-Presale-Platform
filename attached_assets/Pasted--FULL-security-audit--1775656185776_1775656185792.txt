المرحلة 1 — التحقق من الإصلاحات الحالية
━━━━━━━━━━━━━━━━━━

تحقق من endpoints التالية وتأكد هل هي آمنة فعلاً:

1. /api/cron/sync-sol-price
- يجب أن يتطلب Bearer token (CRON_SECRET)
- يجب رفض أي طلب بدون توكن صحيح
- يجب وجود rate limit (3 طلبات كل 5 دقائق)
- تأكد من عدم تسريب private key

2. /api/twitter/timeline/:username
- يجب التحقق من username عبر Regex صارم:
  ^[A-Za-z0-9_]{1,50}$
- رفض أي input غير صالح
- منع SSRF (لا يُسمح بروابط كاملة)
- إضافة rate limit

3. /api/referral/register
- يجب التأكد أن purchaseId موجود في قاعدة البيانات
- يجب أن يكون مرتبط بنفس wallet
- منع تكرار الإحالات
- إضافة rate limit

4. /api/presale/on-chain و /api/sol-price
- يجب وجود rate limiting (60 طلب في الدقيقة)

5. Admin APIs (pagination)
- page يجب أن يكون >= 1
- limit بين 1 و 200 فقط
- منع offset سلبي
- منع queries ضخمة على قاعدة البيانات

المرحلة 2 — أمان الجلسات (Session Security)
━━━━━━━━━━━━━━━━━━

تحقق من نظام المصادقة:

- يجب استخدام req.session.regenerate() بعد تسجيل الدخول OAuth
- منع Session Fixation
- إعدادات الكوكي:
  httpOnly: true
  secure: true (في production)
  sameSite: 'lax' أو 'strict'
- استخدام maxAge فقط وليس expires ثابت
